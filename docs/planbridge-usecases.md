# PlanBridge 유즈케이스 — 실제 데이터 기반 시나리오

---

## 프로젝트 배경

```
프로젝트: FreshMart 어드민 (신선식품 커머스 관리자 시스템)
팀 구성:
  - 기획자: 박지현 (PM, 서비스 기획 담당)
  - 개발자: 김민수 (프론트엔드)
  - 개발자: 이서연 (백엔드)

현재 상태:
  - Next.js 14 App Router 기반
  - 이미 운영 중인 화면: 대시보드, 상품 관리, 주문 관리, 회원 관리
  - PlanBridge 빌드 플러그인(data-pb-id) 적용 완료
  - 초기 스캔 완료 → Oracle DB에 472개 컴포넌트 등록됨
```

---

## 유즈케이스 1: 기존 화면의 정책 등록 (최초 세팅)

### 상황
박지현 PM이 PlanBridge를 처음 도입하고, 기존 "상품 관리" 페이지에 
흩어져 있는 정책들을 시스템에 등록하려 합니다.

### Step by Step

```

  박지현이 Chrome에서 FreshMart 어드민의 상품 관리 페이지를 열고
  PlanBridge 익스텐션을 활성화합니다.

**③ 같은 방식으로 다른 정책도 등록**

```
박지현이 이어서 등록하는 정책들:

상품 가격 입력 필드 (ProductForm.PriceInput)
├─ [VALIDATION] 가격 입력 검증
│   "필수, 숫자만, 100원 이상 10,000,000원 이하, 10원 단위"
│
├─ [UI_SPEC] 가격 표시 포맷
│   "천단위 콤마 자동 적용, '원' 접미사, 입력 시에는 숫자만 표시"
│
└─ [BIZ_RULE] 할인가 제약
    "할인가는 원가의 10% 이상이어야 함 (90% 이상 할인 금지)"

카테고리 선택 (ProductForm.CategorySelect)
├─ [VALIDATION] 카테고리 필수 선택
│   "1차 카테고리 필수, 2차 카테고리는 1차에 따라 조건부 필수"
│
├─ [BIZ_RULE] 카테고리별 필수 정보
│   "신선식품: 유통기한 필수, 냉동식품: 보관방법+해동방법 필수"
│
└─ [DATA_SPEC] 카테고리 API
    "GET /api/categories?parentId={id}, 트리 구조, 최대 3depth"

상품 등록 버튼 (ProductForm.SubmitButton)
├─ [INTERACTION] 등록 버튼 동작
│   "클릭 → 폼 전체 검증 → 확인 모달('등록하시겠습니까?') 
│    → 확인 시 API 호출 → 성공 토스트 → 목록으로 이동
│    → 실패 시 에러 메시지 인라인 표시"
│
└─ [PERMISSION] 상품 등록 권한
    "ROLE_PRODUCT_MANAGER 이상만 등록 가능.
     ROLE_VIEWER는 버튼 자체가 미노출."

── 전역(GLOBAL) 정책도 등록 ──

[GLOBAL] [UI_SPEC] 공통 금액 표시 규칙
  "모든 금액은 천단위 콤마 + '원' 접미사. 0원은 '무료'로 표시.
   음수 금액은 빨간색. 소수점은 버림(floor)."

[GLOBAL] [INTERACTION] 공통 폼 이탈 방지
  "폼 수정 중 페이지 이탈 시 확인 모달 표시.
   '저장하지 않은 변경사항이 있습니다. 이동하시겠습니까?'"

[GLOBAL] [UI_SPEC] 공통 테이블 규칙
  "기본 페이징 20건, 옵션: 20/50/100. 정렬 컬럼 클릭으로 토글.
   빈 데이터 시 '데이터가 없습니다' + 일러스트 표시."
```

## 유즈케이스 2: 기존 정책 수정 (변경 요청)

### 상황
마케팅팀에서 요청이 들어왔습니다: 
"할인율 제한을 90%에서 95%로 올려주세요. 시즌 특가 때 95% 할인이 필요합니다.
단, 95% 할인은 슈퍼관리자만 설정할 수 있게 해주세요."

### Step by Step

**① 박지현이 상품 관리 페이지에서 할인가 입력 필드를 선택**

**③ [저장 + 변경요청 생성] 클릭**

```

   - TITLE: "할인가 제약 정책 변경 (90% → 95%)"
   - DESCRIPTION: (기획자가 입력한 내용)
   - STATUS: AI_PROCESSING

4. AI 분석 호출 → TODO 생성
